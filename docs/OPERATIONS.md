# Операционное руководство

## Логи

- Уровень: `LOG_LEVEL` (trace, debug, info, warn, error, fatal). В проде обычно `info`.
- Файловое логирование: `ENABLE_FILE_LOGGING=true` (если приложение пишет в файлы). Пути и ротация задаются в коде/конфиге приложения.
- Медленные запросы Prisma: `SLOW_QUERY_THRESHOLD_MS`, `SLOW_QUERY_LOGGING`, при отладке — `PRISMA_LOG_QUERIES`.

## Алерты

- Опционально в .env: `ALERT_EMAIL`, `ALERT_WEBHOOK_URL` — для уведомлений по 5xx (если реализовано в коде).

## Бэкапы

- Настроить резервное копирование PostgreSQL (и при необходимости Redis) на прод-сервере (cron, скрипты провайдера или своего скрипта). Частота — по политике (ежедневно минимум).

## После деплоя

- Проверить `GET /api/health` и `GET /api/health/ready`.
- Убедиться, что сайт открывается по домену, нет ошибок CORS при работе с API.
- При изменении схемы БД: выполнять `npx prisma migrate deploy` в процессе деплоя (до или после сборки).
